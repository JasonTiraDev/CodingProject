## üõ†Ô∏è Setup & Requirements

### Technical Requirements
- **Platform**: Any computer with web browser
- **Software**: [p5.js Web Editor](https://editor.p5js.org/) (free, no installation)
- **Internet**: Required for web editor access
- **Alternative**: Local setup with VS Code + Live Server extension

### Classroom Setup
- **Group Size**: 15-25 students (optimal)
- **Equipment**: 1 computer per student or pair programming
- **Display**: Projector/large screen for demonstrations
- **Time**: 2-4 hours per day depending on pace

## üìñ Teaching Strategies

### Recommended Approach
1. **Demo First**: Show working example before explaining code
2. **Incremental Building**: Add one feature at a time
3. **Pair Programming**: Encourage collaboration
4. **Regular Testing**: Run code frequently to catch errors early
5. **Creative Freedom**: Allow students to customize and experiment

### Common Challenges & Solutions

#### Technical Issues
- **Problem**: Code doesn't run
- **Solution**: Check for typos, missing semicolons, bracket matching

- **Problem**: Unexpected behavior
- **Solution**: Use console.log() to debug, step through logic

- **Problem**: Performance issues  
- **Solution**: Optimize loops, limit particle counts

#### Engagement Issues
- **Problem**: Students finish early
- **Solution**: Provide extension challenges, encourage helping others

- **Problem**: Students fall behind
- **Solution**: Pair with advanced students, focus on core concepts

## üìû Support & Troubleshooting

### Quick Reference
- **Syntax Error**: Check for missing brackets, semicolons
- **Reference Error**: Variable not defined or misspelled
- **Game Too Fast/Slow**: Adjust frameRate() or movement speeds
- **Collision Not Working**: Check distance calculations and object sizes

### Getting Help
1. Check console for error messages
2. Compare with working example code
3. Ask classmates or teacher
4. Search p5.js reference documentation
5. Test with simplified version of problematic code
